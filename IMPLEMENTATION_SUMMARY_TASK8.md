# Implementation Summary: Qualitative Error and Model Comparison Analysis

This document verifies the completion of Task 8, providing an automated script to execute qualitative (visual and predictive) breakdown analyses across all model architectures.

## Existing Modules Reused
The analysis script strictly relies on outputs generated by the framework's native inference operations, ensuring no models are actively executed or tampered with during the analysis phase:
- It targets standard prediction `.txt` label files conventionally outputted by the YOLO engine (e.g., `runs/detect/predict/labels/`) to extract detected bounding boxes and confidence thresholds.
- It parses existing ground-truth `.txt` labels to mathematically quantify False Positives and Missed Detections.
- The `sample_images` subdirectories (which often classify data into edge cases like `/night/`, `/occlusion/`, `/dense_traffic/`) are natively used to identify failure patterns.

## New Files Created
- **`scripts/qualitative_analysis.py`**: The main orchestration script. It parses raw YOLO coordinates to determine low-confidence rates, false-positive counts, and identifies the absolute best/worst performing model for any single image natively based on error margins.
- **`paper_outputs/qualitative/comparison_matrix.csv`**: A flattened database assigning the Best Model, Worst Model, Objects Missed, and Average Confidence per analyzed image.
- **`paper_outputs/qualitative/qualitative_summary.md` & `.json`**: The overall qualitative breakdown metrics spanning all images.
- **`paper_outputs/qualitative/QUALITATIVE_ANALYSIS.md`**: A research narrative document detailing common failure patterns isolated directly by folder structures (e.g., "Subsets located in **night** exhibit frequent bounding box omission").
- **`IMPLEMENTATION_SUMMARY_TASK8.md`**: This summary file, verifying constraints.

## Remaining Limitations
- **Image Parsing vs Visualization:** The script calculates "Best Examples" and "Failure Cases" mathematically (by isolating images with 0 Missed/False Positives vs high errors) and copies the raw source images into respective subdirectories (`paper_outputs/qualitative/best_examples/`). However, to see the drawn bounding boxes, the user must still execute the previously generated `scripts/generate_qualitative_figs.py`. This script purely performs the numerical proxy analysis to satisfy the "Do NOT execute inference" rule.
- **Occlusion/Small-Object Deduction:** Because no inference is run and no deep coordinate-overlap IoU mapping is performed (to keep execution lightweight), "small-object failures" or "occlusion" are deduced entirely by whether the user categorized the image inside a `/occlusion/` folder. The script refuses to hallucinate explanations for why an image failed if it isn't categorized.
